# Plan: v2.0 UX Polish

**Date:** 2026-04-13  
**Goal:** Button-up every interactive flow before calling this 2.0. No new features — only finish, fix, and sharpen what's already here.

---

## Audit Summary

The app is architecturally solid. After the v1.x tech-stack audit (Framer Motion removed, UTC bugs fixed, batch API added), the remaining gaps are all UX-layer:

| Gap | Impact | Effort |
|-----|--------|--------|
| Sidebar nav items show no task counts (data already fetched) | HIGH | LOW |
| Deadline invisible on collapsed task row | HIGH | LOW |
| Project section collapse state not persisted (unlike Today) | HIGH | LOW |
| No color picker in project/area create or edit flows | HIGH | MEDIUM |
| Recurrence interval hardcoded to 1 (can't do "every 2 weeks") | HIGH | MEDIUM |
| Overdue tasks show no indicator in Inbox/Upcoming views | MEDIUM | LOW |
| Escape key does not clear bulk selection | MEDIUM | LOW |
| Section drag handle invisible until hover | MEDIUM | LOW |
| Sub-project inline-add doesn't blur-to-submit (inconsistent with every other inline input) | MEDIUM | LOW |
| Area page less polished than project page (no sections, no task reorder) | LOW | HIGH |

**Deferred to v2.1:** in-view search/filter, notes markdown, subtask drag-drop, collaboration.

## Implementation Phases

### Phase 1 — Sidebar Counts & Overdue Indicators (LOW effort, HIGH visibility)

**1a. Wire sidebar task counts**
- `useTaskCounts()` already returns `{ inbox, today, overdue }`. The sidebar renders these counts but only on project/area items.
- Add count badges to **Today** and **Inbox** nav items in `sidebar.tsx`.
- Today badge: show `today` count. If `overdue > 0`, show a separate red dot or merge overdue into the badge with a distinct color.
- Inbox badge: show `inbox` count.

**1b. Overdue task indicator in Upcoming/Inbox**
- `task-item.tsx` already shows deadline urgency in the expanded panel.
- Collapsed row should show a red `⚠` or deadline badge when `task.deadline < today` or `task.whenDate < today` (and task is not completed).
- Already handled in Today view with a red "Overdue" section label — but tasks in Inbox/Upcoming that are past their `whenDate` have no indicator at all.
- Add a `whenDate < today` indicator (small amber/red dot or label) on collapsed row.

**Files:** `packages/web/src/components/ui/sidebar.tsx`, `packages/web/src/components/tasks/task-item.tsx`

---

### Phase 2 — Deadline on Collapsed Row

Currently, deadline is only visible inside the expanded panel. It's the most time-sensitive field and should be surfaced on the collapsed row.

**Changes:**
- In `task-item.tsx`, on the collapsed row metadata strip (where scheduled time, notes indicator, and date already show), add a deadline chip when `task.deadline` is set.
- Show as: `🚩 Apr 20` or `🚩 2d` — small, right-aligned, red if overdue / amber if within 3 days / gray otherwise.
- Match the urgency coloring already used in the expanded panel (`deadline` field display logic).
- Reuse the existing `formatWhenDate()` utility for display.

**Files:** `packages/web/src/components/tasks/task-item.tsx`, `packages/web/src/lib/dates.ts`

---

### Phase 3 — Project Section Collapse Persistence

The Today view persists each time-of-day section's collapsed state in `localStorage` under `todo-today-sections-collapsed`. Project sections use a `isCollapsed` field in the DB via `useUpdateSection` — but changes via the collapse chevron in `section-block.tsx` don't persist to the server (they only update local UI state).

**Changes:**
- Confirm whether `SectionBlock` is calling `useUpdateSection` on collapse toggle, or just doing local state.
- If local-only: wire `updateSection.mutate({ id, isCollapsed: !section.isCollapsed })` on chevron click.
- If already wired but broken: debug why re-load doesn't restore collapsed state.
- The `sections` query already returns `isCollapsed`; just need to initialize the UI state from it.

**Files:** `packages/web/src/components/projects/section-block.tsx`, `packages/web/src/hooks/use-sections.ts`

---

### Phase 4 — Color Picker in Project & Area Flows

Currently colors can be set via API but there's no color picker in the UI (project create/edit, area create/edit). This is a visible gap — the color dot appears in sidebar and project page but users can't choose it.

**Changes:**
- Add a small color swatch picker (8-12 preset colors) to:
  - Sidebar inline project creation input
  - Project page header (click color dot to change)
  - Sidebar inline area creation input
  - Area page header (click color dot to change)
- Presets: same set used anywhere color is referenced — check existing palette in codebase.
- Use a simple `Popover` + grid of color swatches (no third-party library needed).
- Wire to `useUpdateProject` / `useUpdateArea` on select.

**Files:** `packages/web/src/components/layout/sidebar.tsx`, `packages/web/src/app/(views)/project/[id]/page.tsx`, `packages/web/src/app/(views)/area/[id]/page.tsx`

---

### Phase 5 — Recurrence Interval

The DB schema has `recurrence_interval` (integer), stored and returned via API, but the recurrence picker UI only lets users set `recurrenceType` (daily/weekly/monthly/yearly) and `recurrenceMode`. Interval always defaults to 1.

**Changes:**
- In the recurrence popover inside `task-item.tsx`, add a small numeric stepper or input next to the type selector: "Every [N] weeks".
- Validate: integer ≥ 1.
- Pass `recurrenceInterval` in the PATCH call alongside `recurrenceType`.
- Display label on collapsed row (or in expanded): "Every 2 weeks" instead of just "Weekly".
- Schema already supports this — no DB migration needed.

**Files:** `packages/web/src/components/tasks/task-item.tsx` (recurrence popover), `packages/web/src/hooks/use-tasks.ts`

---

### Phase 6 — Small Consistency Fixes

These are individually tiny but collectively meaningful:

**6a. Escape clears bulk selection**
- In `keyboard-provider.tsx`, the Escape handler currently collapses expanded tasks. Add a branch: if `isActive` (selection exists), call `clear()` instead.
- `useSelection` already exposes `clear()` and `isActive`.

**6b. Sub-project blur-to-submit**
- The inline "New project" input in the sidebar doesn't submit on blur (only on Enter/Escape).
- All other inline inputs in the app (sections, tasks, areas) submit on blur.
- Fix: add `onBlur` handler to the project creation input that submits if text is present.

**6c. Section drag handle always faint**
- `section-block.tsx` drag handle is `opacity-0 group-hover:opacity-100`. Change to `opacity-20 group-hover:opacity-100` so there's always a hint it's draggable.

**6d. Recurrence display on collapsed row**
- Tasks with recurrence have no indicator on the collapsed row.
- Add a small ↻ icon (or `♻` label) in the metadata strip, same row as the date/deadline chips.

**Files:** `packages/web/src/components/keyboard/keyboard-provider.tsx`, `packages/web/src/components/layout/sidebar.tsx`, `packages/web/src/components/projects/section-block.tsx`, `packages/web/src/components/tasks/task-item.tsx`

## Out of Scope for v2.0

- Notes markdown rendering
- Subtask reordering / nesting

- Advanced bulk-select (select by filter criteria)
