# v1.1 Scheduling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add scheduled times to tasks ŌĆö `scheduled_time` column, natural language parsing ("meeting at 2pm"), clock badge display, and time-sorted Today sections.

**Architecture:** A `scheduled_time text` column (HH:MM 24h format) is added to tasks. `parseTaskInput` uses chrono-node's `isCertain("hour")` to extract an explicit time from natural language; it co-exists with `whenDate` extraction. Today's section sort function is extended to sort timed tasks before untimed ones. The task item's collapsed row shows a clock badge; the expanded panel shows the time with a click-to-clear button.

**Tech Stack:** Drizzle ORM (sqlite-core), chrono-node, TanStack Query v5, shadcn/ui, Tailwind CSS v4, TypeScript strict

## Task 2: Parser ŌĆö extract scheduledTime from natural language

**Files:**
- Modify: `packages/web/src/lib/parse-task.ts`

The parser already uses chrono-node. When chrono parses "meeting at 2pm tomorrow" it produces a result where `r.start.isCertain("hour")` is `true` (the user explicitly stated an hour). We use this to extract `scheduledTime`. When the user types "tomorrow" with no time, `isCertain("hour")` is `false` (hour is inferred), so we skip it.

- [ ] **Step 1: Add `scheduledTime` to ParseResult**

  In `packages/web/src/lib/parse-task.ts`, change the `ParseResult` interface:

  ```typescript
  export interface ParseResult {
    title: string;
    whenDate?: string;
    timeOfDay?: TimeOfDay;
    /** HH:MM in 24h format */
    scheduledTime?: string;
    deadline?: string;
    isSomeday?: boolean;
    projectId?: string;
    projectName?: string;
  }
  ```

- [ ] **Step 2: Declare `scheduledTime` variable and extract it from chrono results**

  In the same file, in `parseTaskInput`, add `let scheduledTime: string | undefined;` at the top alongside the other `let` declarations:

  ```typescript
  let scheduledTime: string | undefined;
  ```

  Then change the date extraction block (step 5 comment in the code, near line 87) from:

  ```typescript
  // 5. Extract natural-language date from remaining text (skip if someday)
  const dateResults = chrono.parse(text, new Date(), { forwardDate: true });
  if (dateResults.length > 0) {
    const r = dateResults[0];
    whenDate = r.date().toISOString().slice(0, 10);
    text = text.slice(0, r.index) + text.slice(r.index + r.text.length);
  }
  ```

  To:

  ```typescript
  // 5. Extract natural-language date (and time if explicitly stated) from remaining text
  if (!isSomeday) {
    const dateResults = chrono.parse(text, new Date(), { forwardDate: true });
    if (dateResults.length > 0) {
      const r = dateResults[0];
      whenDate = r.date().toISOString().slice(0, 10);
      // Only extract a scheduled time if the user explicitly stated one (not inferred)
      if (r.start.isCertain("hour")) {
        const h = String(r.start.get("hour")).padStart(2, "0");
        const m = String(r.start.get("minute") ?? 0).padStart(2, "0");
        scheduledTime = `${h}:${m}`;
      }
      text = text.slice(0, r.index) + text.slice(r.index + r.text.length);
    }
  }
  ```

  Note: the original code already skipped date parsing when `isSomeday` was set (it still parsed). This version wraps it in `if (!isSomeday)` to be explicit and consistent.

- [ ] **Step 3: Include `scheduledTime` in the return value**

## Task 5: Today view ŌĆö sort by scheduled time within sections

**Files:**

Currently `tasksBySection` just filters. We sort timed tasks to the top of each section, earliest first. Tasks with no scheduled time appear after timed ones.

- [ ] **Step 1: Update `tasksBySection` to sort by scheduledTime**

  In `packages/web/src/app/(views)/today/page.tsx`, change the `tasksBySection` function from:

  ```typescript
  function tasksBySection(sectionId: TimeOfDay | null): Task[] {
    return activeTasks.filter((t) => (t.timeOfDay ?? null) === sectionId);
  }
  ```

  To:

  ```typescript
  function tasksBySection(sectionId: TimeOfDay | null): Task[] {
    return activeTasks
      .filter((t) => (t.timeOfDay ?? null) === sectionId)
      .sort((a, b) => {
        if (!a.scheduledTime && !b.scheduledTime) return 0;
        if (!a.scheduledTime) return 1;
        if (!b.scheduledTime) return -1;
        return a.scheduledTime.localeCompare(b.scheduledTime);
      });
  }
  ```

  `localeCompare` works correctly on "HH:MM" strings since they're zero-padded and lexicographic order matches chronological order.
