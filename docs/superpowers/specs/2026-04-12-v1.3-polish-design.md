# v1.3 Polish ŌĆö Design Spec

**Date:** 2026-04-12
**Status:** Approved

---

## Overview

Five small, additive quality-of-life improvements that ship as a single release. No breaking changes. One new schema-free API route, one new API endpoint, and pure UI work for the rest.

---

## Features

### 1. Sidebar Task Count Badges

Display live task counts next to **Inbox** and **Today** in the sidebar nav.

**API:** `GET /api/tasks/counts` ŌĆö new route, returns `{ inbox: number, today: number, overdue: number }` via three `COUNT(*)` SQL queries. Same Bearer auth as all other routes.

- `inbox`: `when_date IS NULL AND project_id IS NULL AND area_id IS NULL AND parent_task_id IS NULL AND is_completed=0 AND is_cancelled=0 AND deleted_at IS NULL AND is_someday=0`
- `today`: `when_date = today AND parent_task_id IS NULL AND is_completed=0 AND is_cancelled=0 AND deleted_at IS NULL`
- `overdue`: `when_date < today AND parent_task_id IS NULL AND is_completed=0 AND is_cancelled=0 AND deleted_at IS NULL`

**Hook:** `useTaskCounts()` in `use-tasks.ts` ŌĆö `queryKey: ["task-counts"]`, `staleTime: 60_000`. Mounted once in the sidebar.

**Display:** Small muted pill (shadcn Badge, secondary variant, `text-xs`) inline after the label. Hidden when count is 0. Only Inbox and Today show badges ŌĆö overdue count is consumed by the Today page (see Feature 2).

---

### 2. Overdue Group in Today

Surface tasks past their `when_date` in a dedicated section at the top of the Today view.

**API:** Add `overdue` filter case to `GET /api/tasks`:
```
when_date < today
AND parent_task_id IS NULL
AND is_completed = 0
AND is_cancelled = 0
AND deleted_at IS NULL
```

**Today page:** Add `useTasks("overdue")` query alongside existing `useTasks("today")` and `useTasks("completed_today")`. When overdue tasks exist, render an "Overdue" section above Morning / Day / Night.

**Visual treatment:** Destructive-tinted left border and section heading. Each task shows its `when_date` to indicate how far overdue it is. Section is **not collapsible** ŌĆö overdue tasks must always be visible.

**Interaction:** Moving an overdue task with `T` (or context menu ŌåÆ Today) sets `when_date = today`, removing it from Overdue and placing it in the appropriate time-of-day group.

---

### 3. Duplicate Task

Clone a task in full via context menu.

**API:** `POST /api/tasks/[id]/duplicate` ŌĆö fetches original, clones all fields:
- Carried over: `title`, `notes`, `projectId`, `areaId`, `sectionId`, `whenDate`, `scheduledTime`, `timeOfDay`, `deadline`, `recurrenceType`, `recurrenceMode`, `recurrenceInterval`, `recurrenceEndsAt`, `isSomeday`
- Reset: `isCompleted = false`, `completedAt = null`, `isCancelled = false`, `deletedAt = null`, `parentTaskId = null`
- New: `id` (nanoid), `createdAt`, `updatedAt`
- Position: average of original and its next sibling's position values (fractional index insert after original); if no next sibling, `position = original.position + 1.0`

**Subtasks:** Not duplicated. Clone starts with no subtasks.

**Hook:** `useDuplicateTask()` in `use-tasks.ts` ŌĆö mutation POSTs to `/api/tasks/[id]/duplicate`, invalidates relevant task list query on success.

**UI:** "Duplicate" context menu item in `task-item.tsx`, placed between the Cancel toggle and the Delete separator.

---

### 4. "Move to Next Week" Shortcut

Move the focused task to `today + 7 days` via keyboard or context menu.

**Shortcut definition** (added to `SHORTCUT_DEFS` in `shortcut-config.ts`):
```ts
{ id: "task-move-next-week", defaultKey: "w", label: "Move to Next Week", category: "task-actions" }
```

**Handler** in `task-list.tsx` via `useShortcutAction("task-move-next-week", ...)`:
- Sets `whenDate = today + 7 days`, clears `isSomeday = false`
- Snapshot-based undo toast (same pattern as existing T/I/S handlers)
- No-op when `focusedTaskId` is null

**Context menu:** "Move to Next Week" added to `task-item.tsx` context menu alongside Today / Inbox / Upcoming / Someday move items.

**Keyboard overlay and settings page** pick up the new shortcut automatically via `SHORTCUT_DEFS`.

---

### 5. Collapsible Today Sections

Allow Morning / Day / Night sections in Today to be individually collapsed, with state persisted to localStorage.

**Persistence:** `localStorage["todo-today-sections-collapsed"]` stores `{ morning: boolean, day: boolean, night: boolean }`. Default: all `false` (expanded). Initialized via lazy `useState` initializer on mount.

**UI:** `ChevronDown` / `ChevronRight` icon button on the right side of each section header. Clicking toggles that section and writes to localStorage. Collapsed task list animates out via Framer Motion (same collapse animation used for task expand/collapse).

**Collapsed heading:** Shows section title + a subtle task count ("3 tasks") so the user knows content is hidden.

**Empty sections:** Already hidden regardless of collapse state (no change to existing behavior).
